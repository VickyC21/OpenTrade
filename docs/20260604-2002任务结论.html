实施过程中的主要问题与处理

1. 结构拆分后旧测试 patch 目标失效 原测试仍 patch opentrade.backends.providers.*，而真实执行路径已经转到拆分后的 backend 模块，导致测试触发真实网络访问。 处理方式是把测试 patch 目标切换到 provider_akshare、provider_efinance、provider_yfinance 的真实实现点。
2. `provider_akshare.py` 存在机械拆分遗留缺失导入 `_build_akshare_retry_policy()` 使用了 ProviderContractError，但模块初始未导入。 该问题已补齐，并通过相关结构测试验证。
3. shared `quote.*` 语义原本不真实 东财 quote_id 与 Yahoo ticker 原本被混放在同一 shared 字段中，既不统一也不可静态验证。 本次改为共享行情标识语义，并把 provider-native ID 留在 adapter 内部翻译或明确拒绝。
4. PowerShell 编辑时需特别注意中文和换行 过程中曾出现一次把字面量 `r`n 误写进 Python 文件的情况，立即回查并修复。 最终所有写回均使用 UTF-8 无 BOM，且已通过 Python 编译与测试验证。
